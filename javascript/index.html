Histórico

Uma coisa que quem estuda tecnologia está cansado de saber é que tão importante quanto cair de cara na prática e sair codificando, é estudar a história de uma tecnologia e os conceitos.

Vamos então explanar conhecimentos sobre essa linguagem cedosa por meio de topicos:

  • Essa linguagem foi especificamente criada para navegadores web, especificamente o navegador Netscape da empresa Netscape (foda né mamrketeiro). Bem de forma resumida por volta de 1996, o quê os programadores tinham na web é que eles utilizavam a web para compartinharem informaçoes de forma mais estaticas no html, até que alguem achou aquilo sem graça e começou a pensar em criar uma linguagem que pudesse interargir com elementos do html e moficar em tempo real ou no caso fazer a validação de algo quase que em tempo real, não sei de quem foi a ideia Foda-se mas o Brendan Eich que foi o cara que criou a linguagem e com foco de resolver esse problema validar formularios.

  • Essa linguagem então nasceu com objetivo claro e é uma linguagem de programação interpretada: Ou seja seu código fonte é executado por um programa de computador denominado interpretador e o que esse danado de interpretador traria de beneficio?
    1. Uma vez interpretada pelos navegadores, os mesmos tornaria possivel manipular estruturas dos html e tals
    2. Outros interpretadores como Node.js que no tempo nem sonhava em ser criado, mais tarde seria criado, tornando a linguagem popular e com maior capacidade de resolução de problemas

  • O js é uma linguagem de alto nível: tem entendimento mais próximo da realidade, essa característica geralmente define se ela é Orientada a Objetos (OO) ou não, e ela é multi paradigma: "lembrando que 1 paradigma é uma forma de classificar determinada linguagem com base no seu funcionamento", logo se você pensou, então se ela é multi paradigma, ela consegue funcionar de 2 formas. De fato é isso! O js utiliza tanto do paradigma OO quanto funcional.

  • E aí em que isso afeta? Afeta que podemos usar o melhor dos 2 mundos o "OO" que por padrão as variáveis são mutáveis e o paradigma funcional onde geralmente as linguagens que tem essa característica de serem funcional tem variáveis imutáveis . Podemos então para exemplificar construir algo no paradigma funcional e no OO:
    *A OO determina que o código deve ser modelado de forma a se aproximar ao mundo real, e que o mesmo em execução no computador tenha tal modelagem representada por uma estrutura de objetos, características e ações. Tais objetos podem interagir uns com os outros modificando o estado do programa até que resultem nas saídas esperadas. Para viabilizar essa estrutura, OO propõe o uso de Classes, Atributos e Métodos, unido a características como Abstração, Encapsulamento, Herança e Polimorfismo. Os objetos são instâncias das classes e possuem um estado durante a execução do software podendo as mesmas serem modificadas, persistidas e recarregadas de acordo com a lógica implementada. 
                    
                    Object.defineProperty(this, "funcao", {
                        enumerable: true,
                        value: funcao,
                        writable: false,
                      });

    * O paradigma funcional é derivado do declarativo, onde o objetivo é declarar ao computador o resultado esperado, e não o passo a passo para construção deste resultado. Essa forma de compor o código traz duas características que a tornam bastante diferente das demais, que são a imutabilidade e por consequência a não mudança de estado do software. 
                    
                    const funcao = (nota1,nota2) => nota1+nota2
  • E ai o que danado essas caracteristicas nos tras de bom ? Bem essa caracteristica tras coisas muitos boas para a linguagem por que na verdade o fato do javascript ser uma linguagem fracamente tipada acabou que ocasionava muitos erros e dores de cabeça aos programadores que passavam a maior parte do tempo debugando codigo em vez de programando, logo alguem percebeu esse problema e novamente caimos na mesma tecla de que programadores resolvem problemas, e então surgiu o typescript uma extenção do js ou superset do javascript que acrescenta typagem estatica a linguagem e alem disso novos recursos, e isso tudo graça a caracteristica do paradigma funcional que garante a imutabilidade do tipo de dados, esse tipo de dados é nada mais que uma interface provida com essa extenção do js, o typescript.
